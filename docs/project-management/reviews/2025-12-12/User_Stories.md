# User Stories
**Project:** Ticket Masala
**Date:** 2025-12-12

## 1. Customer Stories
*   **US-101:** As a **Customer**, I want to **create a new ticket** so that I can report an issue or request a service.
*   **US-102:** As a **Customer**, I want to **view the status of my tickets** so that I know when they will be resolved.
*   **US-103:** As a **Customer**, I want to **add comments to a ticket** so that I can provide additional details or answer questions.
*   **US-104:** As a **Customer**, I want to **receive email notifications** when my ticket status changes.

## 2. Agent Stories
*   **US-201:** As an **Agent**, I want to **view a dashboard of assigned tickets** so that I can prioritize my daily work.
*   **US-202:** As an **Agent**, I want to **log time/effort** on a ticket so that the project costs are accurate.
*   **US-203:** As an **Agent**, I want to **reassign a ticket** if it is not in my domain of expertise.
*   **US-204:** As an **Agent**, I want to **see AI recommendations for similar tickets** so that I can solve issues faster.

## 3. Manager Stories
*   **US-301:** As a **Manager**, I want to **batch assign new tickets** to available agents so that I can clear the backlog quickly.
*   **US-302:** As a **Manager**, I want to **view team workload reports** so that I can prevent burnout or bottlenecks.
*   **US-303:** As a **Manager**, I want to **approve or reject completed tickets** so that quality is maintained before closing.

## 4. Admin Stories
*   **US-401:** As an **Admin**, I want to **configure custom fields** for different domains (e.g., IT vs. HR) so that the system captures relevant data.
*   **US-402:** As an **Admin**, I want to **manage user roles and permissions** so that data security is upheld.
